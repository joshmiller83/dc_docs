One of the more important line items to your Drupal Commerce store is a product line item. By default, there is only one type of Line Item defined. There can be others, but unlike Product Entities or Customer Profile Entities, these kinds of entities don't mean much unless they have some custom API code to give them more purpose.

Product line items inherit normal line item properties:

  • Line Item ID
  • Order ID
  • Product SKU (specific to Product line items)
  • Type
  • Label
  • Quantity
  • Created
  • Changed
  • Data

Each line item type can also be treated just like any other entity type, you can add fields to it and play with it in Views. For example, it's possible to add a field to your default product line item called "coupon" and then create a rule that will discount the line item if the value of that field equals one of your pre-defined coupons. This would also be very handy for tracking advertising because you could call up with Views all of your orders that had a specific coupon applied without very much knowledge about databases.
